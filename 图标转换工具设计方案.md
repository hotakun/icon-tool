# 图标转换工具 — 设计方案

> 将 PNG/JPG 图片批量转换为多尺寸 ICO 图标文件，支持透明度、圆角调节、PWA 离线使用。

---

## 一、整体定位

一个**纯前端**的图标生成工具。用户在浏览器中上传一张图片，调整圆角，勾选想要的尺寸，一键生成包含全部选中尺寸的 `.ico` 文件。

**纯前端意味着**：不需要服务器、不需要上传到云端、图片处理全部在本地浏览器完成，隐私安全。

---

## 二、核心功能一览

| 功能 | 说明 |
|------|------|
| 拖放/选择上传 | 支持 PNG、JPG、WebP、SVG 等浏览器能打开的图片格式 |
| 实时预览 | 上传后以大尺寸展示处理后的效果（带圆角） |
| 小尺寸预览矩阵 | 同时展示 16/32/48/64/128/256/512 的缩略预览 |
| 圆角调节 | 滑块控制，0% ~ 50%，对应 0px ~ 最大半径 |
| 尺寸勾选 | 6 个尺寸复选框（16/32/64/128/256/512），默认全选 |
| 圆角独立勾选 | 可选择「仅大尺寸加圆角，小尺寸保持直角」 |
| 一键生成 ICO | 将所有选中尺寸打包为一个 .ico 文件 |
| 单独下载 PNG | 也可选择单独下载某个尺寸的 PNG |
| 拖放重新上传 | 随时拖一张新图进来替换 |

## 三、ICO 格式原理（技术核心）

一个 `.ico` 文件本质上是一组 PNG 图片的容器：

```
┌──────────────────────┐
│  ICO 文件头 (6字节)    │  ← 标记这是 ICO + 包含几张图
├──────────────────────┤
│  目录项 × N (16字节×N) │  ← 每张图：尺寸、位置、大小
├──────────────────────┤
│  PNG 数据 (16×16)     │  ← 就是一张完整的 PNG 文件
├──────────────────────┤
│  PNG 数据 (32×32)     │
├──────────────────────┤
│  PNG 数据 (64×64)     │
│  ...                 │
└──────────────────────┘
```

**关键点**：
- Windows 从 Vista 开始，ICO 内部直接用 PNG 存储（支持 32 位 RGBA 全透明通道）
- 我们只需要：Canvas 处理图片 → 加圆角 → 缩放到各尺寸 → 导出 PNG blob → 拼接成 ICO
- 完全不依赖任何后端，Canvas API 足够完成全部工作

## 四、核心算法

### 4.1 圆角裁切

```
1. 创建目标尺寸的离屏 Canvas
2. 用 roundRect() 绘制圆角矩形路径
3. ctx.clip() 设置为裁剪区域
4. ctx.drawImage() 绘制缩放后的原图
5. 圆角区域外的像素自动变为透明
```

### 4.2 ICO 文件拼接（纯二进制操作）

```
步骤：
1. 对每个选中尺寸，先生成带圆角的 PNG blob
2. 用 FileReader 读取为 ArrayBuffer
3. 计算 ICO 文件头（6 字节）
4. 为每张图写入目录项（16 字节）
5. 按顺序拼接所有 PNG 的原始数据
6. 创建一个 Blob，MIME 设为 image/x-icon
7. 触发浏览器下载
```

### 4.3 圆角百分比 → 实际像素的计算

```
圆角像素 = min(图片宽, 图片高) × 百分比 / 100 × 0.5

例如：256×256 的图，圆角 50%
→ 圆角半径 = 256 × 50% × 0.5 = 64px
→ Canvas roundRect(x, y, 256, 256, 64)
```

### 交互细节

- **未上传图片时**：预览区显示虚线框 + "点击或拖放图片"提示
- **上传后**：实时显示处理效果，圆角滑块拖动时所有预览同步更新
- **生成按钮**：点击后 6 个尺寸同时处理，完成后触发下载
- **手机端**：支持从相册选图、拍照上传

## 六、技术栈

| 层面 | 方案 |
|------|------|
| 界面 | 纯 HTML + CSS + JavaScript（单文件） |
| 图片处理 | Canvas 2D API（clip + drawImage + scale） |
| ICO 拼接 | 手动构建二进制（ArrayBuffer + DataView） |
| 下载 | Blob + URL.createObjectURL + a.click() |
| PWA | manifest.json + Service Worker（与音乐播放器相同方案） |

---

## 七、边缘情况处理

| 场景 | 处理方式 |
|------|---------|
| 上传非正方形图片 | 居中裁切为正方形（取短边） |
| 上传过小的图片（< 16px） | 提示图片太小，建议至少 512×512 |
| 超大图片（> 4096px） | 自动降采样到 2048px 再处理 |
| 未勾选任何尺寸 | 按钮置灰，提示至少选一个 |
| 浏览器不支持 Canvas | 显示提示文字 |
| 纯 JPG 无透明通道 | 圆角区域显示为透明（自动处理） |
| 移动端使用 | 完全响应式，触控友好 |
